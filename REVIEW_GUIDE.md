# Review Pull Requests

This file demonstrates pull request review skills.

## What Makes a Good PR Review?

1. **Check functionality** — Does the code do what it claims?
2. **Review style** — Is the code clean and consistent?
3. **Test coverage** — Are there appropriate tests?
4. **Documentation** — Is it documented clearly?

## PR Review Checklist

- [ ] Code is self-explanatory or well-commented
- [ ] No unnecessary complexity
- [ ] Edge cases handled
- [ ] No security vulnerabilities
- [ ] Performance considered
